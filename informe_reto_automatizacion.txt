1.1 ANALISIS Y PLANIFICACION
-----------------------------
Se analizo el sitio web https://www.bancolombia.com/personas identificando
que es un portal Liferay con componentes Vue/React que renderizan el contenido
dinamicamente.

Stack tecnologico definido:
  - Java 17 + Gradle 9.5.1
  - Serenity BDD 5.3.9 (framework de automatizacion con reportes)
  - Cucumber 7.34.3 (escenarios Gherkin)
  - Screenplay Pattern (diseno mantenible)
  - PDFBox 3.0.7 (validacion de PDFs)
  - JUnit Platform 5.11.4 (ejecutor de pruebas)
  - Hamcrest 3.0, SLF4J 2.0.18

1.4 FLUJO DE AUTOMATIZACION
----------------------------

  Feature: Validacion de documentos de inversion virtual Bancolombia
    Scenario: Descargar y validar el Reglamento de Inversion Virtual
      Given el usuario se encuentra en la pagina de personas de Bancolombia
      When navega al menu "Productos & Servicios"
      And selecciona la opcion "Inversiones"
      And selecciona la opcion "Inversion Virtual"
      And hace clic en "Documentos"
      And descarga el documento "Reglamento Inversion Virtual Bancolombia"
      Then el documento PDF debe ser el correcto

1.5 EXPLICACION TECNICA DE CADA PASO
--------------------------------------

PASO 1: Navegar a la pagina de Personas
  - Task: NavigateTo -> Open.url(Urls.PERSONAS_PAGE)
  - Usa constante Urls.PERSONAS_PAGE en vez de string literal.

PASO 2: Seleccionar "Productos & Servicios"
  - Task: SelectFromMenu.menuProductosServicios()
  - JavaScriptClick.on() para evitar overlay bc-modal-overlay.
  - Selector via BancolombiaPage.MENU_PRODUCTOS_SERVICIOS (usa Selectors.MENU_PRODUCTOS_ID).

PASO 3: Seleccionar "Inversiones"
  - SelectFromMenu.option(Labels.INVERSIONES)
  - Mapeado en NAVIGATION_ACTIONS via Strategy Pattern.

PASO 4: Navegar a "Inversion Virtual"
  - Open.url(Urls.INVERSION_VIRTUAL_PAGE) + HideOverlay.now()
  - HideOverlay ejecuta Scripts.HIDE_OVERLAY (Evaluate.javascript).

PASO 5: Hacer clic en "Documentos" (Acordeon)
  - ExpandAccordion.documentosSection() -> ClickAccordionByLabel.withText(Labels.DOCUMENTOS)
  - La Interaction ClickAccordionByLabel busca elementos con el texto exacto
    y dispara dispatchEvent(new MouseEvent('click')).
  - Los scripts JS estan en Scripts.java con metodos generadores.

PASO 6: Descargar el PDF
  - DownloadDocument.reglamentoInversionVirtual() -> Open.url(Urls.REGLAMENTO_PDF)
  - URL centralizada en Urls.REGLAMENTO_PDF.

PASO 7: Validar el PDF
  - Question: ThePDF.isCorrect()
  - Valida metadatos: titulo contiene "reglamento" o autor contiene "bancolombia"
  - Fallback a extraccion de texto con PDFTextStripper.
  - Cadenas en Labels.PDF_TITLE_MARKER y Labels.PDF_AUTHOR_MARKER.

1.6 DECISIONES TECNICAS IMPORTANTES
-------------------------------------

a) SERENITY 5.3.9 + CUCUMBER 7:
   - serenity-cucumber 5.3.9, JUnit Platform (@Suite + @IncludeEngines("cucumber")).
   - No se usa el deprecated @RunWith(CucumberWithSerenity.class).

b) SCREENPLAY PATTERN:
   - Interactions: operaciones reutilizables (HideOverlay, ClickAccordionByLabel)
   - Tasks: orquestan Interactions (NavigateTo, SelectFromMenu)
   - Questions: validaciones (ThePDF)
   - Page Objects: localizadores (BancolombiaPage)

c) PRINCIPIOS SOLID:
   - SRP: cada clase hace una sola cosa
   - OCP: Map<String,Runnable> en vez de if/else en step definitions
   - DIP: Tasks/Interactions dependen de Performable (abstraccion Serenity)

d) STRATEGY PATTERN:
   Step definitions sin condicionales. Cada opcion se mapea en Map<String,Runnable>:
     NAVIGATION_ACTIONS = Map.of(
       Labels.INVERSIONES,       -> SelectFromMenu.option(Labels.INVERSIONES)
       Labels.INVERSION_VIRTUAL, -> Open.url(...) + HideOverlay.now()
     )

e) ELIMINACION DE MAGIC-STRINGS:
   Todos los strings en constants/:
   - Urls: URLs del sitio y PDF
   - Labels: nombres de UI y textos de validacion
   - Selectors: IDs, clases, XPath templates con String.format
   - Scripts: JS con %s placeholders, metodos generadores
   - Messages: logs y errores (formato {0} para el actor)

f) MANEJO DEL OVERLAY:
   bc-modal-overlay bloquea clicks. HideOverlay lo oculta con Evaluate.javascript.
   Para el acordeon: dispatchEvent(new MouseEvent('click')) via ClickAccordionByLabel.

g) VALIDACION DE PDF:
   Metadatos (titulo + autor) con case-insensitive matching.
   Fallback a extraccion de texto con PDFTextStripper.

h) CONFIGURACION DE NAVEGADOR:
   Chrome headless, 1920x1080, autodownload=true, screenshots FOR_FAILURES.
